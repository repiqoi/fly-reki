# راهنمای راه‌اندازی XHttp روی سرویس Dockfly و render

این راهنما مراحل راه‌اندازی یک سرویس XHttp ساده بر روی سرویسRender / Dockfly رو توضیح می‌ده.

---

## مرحله 1: فورک کردن پروژه
- پروژه رو در گیتهاب فورک کنید روی حساب کاربری خودتون.

---

## مرحله 2: وارد سایت Dockfly شوید
Render.com
- به سایت [https://dockfly.app/](https://dockfly.app/) برید.
- وارد حساب کاربری خودتون بشید یا اکانت بسازید.

---

## مرحله 3: ساخت پروژه جدید
- در قسمت **Projects**، گزینه **New Project** رو بزنید.
- یک اسم دلخواه وارد کنید و **Create** رو کلیک کنید.

---

## مرحله 4: افزودن سرویس اول (GitHub)
- در صفحه پروژه، روی **Add First Service** کلیک کنید.
- گزینه **GitHub** رو انتخاب کنید.
- منتظر بمونید تا لیست ریپازیتوری‌ها بیاد بالا.
- پروژه‌ای که فورک کردید (مثلاً `fly-reki`) رو انتخاب کنید.
- در صفحه **Configure Service**، گزینه **Web** → **Medium** رو انتخاب کنید و **Create** رو بزنید.
- حالا روی پروژه ساخته شده کلیک کنید تا صفحه مدیریت باز بشه.

---

## مرحله 5: تنظیم متغیرها
- از منوی بالا روی **Variables** کلیک کنید.
- مقادیر زیر رو وارد کنید:

| نام متغیر | مقدار |
| --------- | ------- |
| `RELAY_PATH` | `/xhttp` (مقدار دلخواه خودتون) |
| `TARGET_HOST` | `yourdomain.io` (دامین سرور شما) |
| `TARGET_PORT` | `443` |

- سپس روی **Deploy** کلیک کنید.
- آدرس لینک دریافتی روexample.dockfly.app در قسمت **Overview** قرار داره و به عنوان **sni-host-address** در کانفیگ xhttp سرور استفاده کنید..
---
برای رندر میتونید دامین شخصیتون رو بذارید و بجای دامین پروژه ای که ساختین استفاده کنید.
